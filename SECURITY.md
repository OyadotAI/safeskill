# Security policy

## Supported versions

We prioritize security fixes on the default development branch (`main`) and
released packages as applicable. If you need clarification for a specific
version or npm package, mention it in your report.

## Reporting a vulnerability

**Please do not** report security vulnerabilities through public GitHub issues,
discussions, or pull requests.

Preferred options:

1. **GitHub** — If [private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability)
   is enabled for this repository, use **Security → Report a vulnerability** on
   the repo.
2. **Maintainers** — If GitHub reporting is not available, contact the project
   maintainers through a private channel (for example the organization behind
   [Oya.ai](https://oya.ai) / [safeskill.dev](https://safeskill.dev)).

Include as much detail as you can safely share: affected component, steps to
reproduce, impact, and any suggested fix or patch.

## What to expect

- We will acknowledge receipt when we can and work toward a fix on a timeline
  appropriate to severity.
- We may ask follow-up questions or coordinate disclosure once a mitigation is
  ready.

Thank you for helping keep SafeSkill users safe.
